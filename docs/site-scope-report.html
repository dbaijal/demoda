  • Promotional Landing template (3,671 pages)

Key Architectural Notes

1. AEM Responsive Grid - All pages use AEM's 12-column responsive grid (aem-Grid--default--12, tablet, phone breakpoints)

2. Kameleon Design System (kmd-) - Product catalog cards use a dynamic design system for real-time product data display

3. Experience Fragments - Shared content (header, footer, promotional banners) leverages AEM Experience Fragments for reuse

4. Anchor Navigation - Product/category pages use data-anchor-nav for in-page sticky navigation

5. Spacing System - Consistent utility classes: mt-l1 to mt-l5, p-l1 to p-l5 (16px increments)

6. Virtual Tour Pages - Unique 3D viewer technology requiring special embed/integration handling

7. Recommended Migration Approach

PhaseActionRationale
Phase 1 Build shared components: hero, textimage (all 7 variants), CTA (all 8 styles), container These blocks are used across all template types - build once, reuse everywhere
Phase 2 Product Category Landing + Form/Lead-Gen templates Covers 72% of pages; highest-traffic, highest-value content
Phase 3 Learning Center + Promotional Landing templates Content marketing and campaign pages - high engagement value
Phase 4 Event + Virtual Tour templates Medium volume, specialized interactions (3D viewers, registration)
Phase 5 Remaining Tier 3 templates (Blog, Brand, About, CSR, etc.) Low volume, many custom one-off components; handle last
Special Cell Line Database - consider spreadsheet/data-driven approach in EDS Structured data with 621 near-identical pages; template + data source is more efficient
