6. Design System & Content Patterns

Key Visual Patterns

  1. Cards-first design — textimage teaser variants are the most-used component, mapping to 10+ card variants in EDS
  2. Hero + Sticky Anchor Nav + Sectioned Cards — Standard product page pattern used by ~35,000 pages
  3. Section colored backgrounds — Light blue "wells" and dark navy sections for visual rhythm
  4. Dual CTA pattern — Primary (solid brand red) + Secondary (outline/link with right-arrow icon)
  5. Mondrian promotional tiles — Side-by-side offer cards with background images + text overlay
  6. Left sidebar taxonomy nav — Persistent on all product/category pages, auto-generated from page tree
  7. Dynamic product data cards — Kameleon (kmd-*) cards powered by product catalog API

9. Risk Factors & Considerations

RiskImpactMitigation
Dynamic product cards (kmd-*) require API integration High Custom EDS block with fetch() to product catalog API; may need backend proxy
Kameleon Design System is proprietary Medium Reverse-engineer card rendering logic or negotiate API access documentation
89K URLs includes significant dead content Low Pre-filter with HTTP status checks; focus on active pages only
Left sidebar nav is taxonomy-driven Medium Build auto-navigation block reading from EDS page index/query-index
Personalization (Adobe Target) content Medium Decide scope: drop personalization or implement EDS experimentation framework
Form submissions routed to Marketo Medium Map Marketo endpoint configuration for any active forms requiring migration
Brightcove video IDs may expire Low Verify video availability during content migration; flag broken embeds
Large page count (~61K) extends content migration timeline High Prioritize by traffic/business value; automate with import scripts per template
Key decision required: The dynamic product cards (kmd-*) appear on high-traffic product pages. These pull live product data from an API. The team needs to decide whether to (a) replicate the dynamic behavior with a custom EDS block + API, (b) pre-render static snapshots during import, or (c) exclude product catalog sections from migration scope.
